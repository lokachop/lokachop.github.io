Ripping UE5 is Annoying
22/03/2025

Ripping audio from UE5 is a chore in comparison to other games, and for no apparent reason, let's run a process comparison

    Unity

    1. Download AssetRipper
    2. Open AssetRipper
    3. Open the game folder and drag the [GameName]_Data folder to AssetRipper
    4. Click Export -> Export all Files
    5. Select a folder to export to
    6. Done!

    Godot

    1. Download Godot PCK Explorer
    2. Open Godot PCK Explorer
    3. Drag the game .exe or game .pck
    4. Click Extract -> Extract All
    5. Select a folder to export to
    6. Done!

    Meanwhile, Unreal Engine 5+

    1. Download FModel
    2. Open FModel
    3. Click Directory -> Selector
    4. Be lucky and have your game preconfigured, otherwise click Add Undetected Game, Give it a name and the game path
    5. Pray it guesses the UE version otherwise somehow figure out the version of UE5 the game is using, and set it on UE Versions
      6. It is very likely that the game is encrypted, so we have to obtain the AES encryption key
    6. Be lucky and get the key by googling the name of your game, otherwise use one of the many sketchy tools to extract it
    7. We now need to obtain the mappings, which FModel seems to need to be able to rip anything, Download Dumper-7
    8. Open up your game, and inject Dumper-7 with your favourite injector
    9. Go to FModel, Settings -> Mapping File Patch and select the mapping file which Dumper-7 generated defaults to C:/Dumper-7/
    10. We can finally start ripping, select the .utoc file on Archives
    11. Go to folders, GameName, Audio, and open a folder to find sounds to rip
    12. Go to packages and select the ones you want to rip
    13. Right click -> Extract in New Tab
    14. On the window that opens, right click the file list and click Save Playlist
    15. Find where the files get exported to by clicking on the console output
    16. You will get .binka files, Convert them to .wav using a sketchy program from some random GitHub thread, that also needed UE5 source code to compile
    17. Realize the program emits broken .wav files and write a python ffmpeg handler to repair them
    18. Done!
